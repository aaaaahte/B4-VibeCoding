# B4-VibeCoding

在开始 Vibe Coding 之前，先为 AI 准备一份**可执行的开发指南**。

> **B4-VibeCoding（B4VC）** 的目标：把「一个简单但有创意的 idea」逐步补全成**可落地的需求与工程方案**，让 Coding Agent 能按图施工，**make idea come true**。

## 为什么做 B4-VibeCoding？

作为零编程基础的 Vibe Coding 选手，我们可能并不熟悉软件设计与开发的基础知识：不会做需求分析、不了解技术架构、搞不清 API/联调/测试。  
**但我们拥有一个简单、实用、并且创意十足的 idea。**

## B4VC 会做什么？

1. **需求文档（PRD）**  
   用户输入 idea，B4VC 会从产品经理视角追问关键细节，逐步补齐需求。输出结构经过优化的 Markdown 文档，便于 Coding Agent 理解与执行。

2. **技术架构设计**  
   基于需求文档，B4VC 主动完成技术选型与架构设计，包括但不限于：  
   - 技术选型：前端 / 后端 / 数据库 / 中间件  
   - 系统架构：单体 vs 微服务、云服务器 vs 容器化、是否需要负载均衡、第三方依赖接入  
   - 数据库设计：ER 图、字段定义、关系、索引  
   - 前端架构：目录结构、状态管理、路由规划、权限控制

3. **API 接口设计**  
   梳理接口清单并定义接口细节：URL、请求方式、请求参数、返回数据、错误码，以及 Mock 数据方案，交付完整的接口列表文档。

4. **任务清单（开发计划）**  
   以 Scrum Master 视角拆分开发任务、设定优先级；每个任务给出明确交付物与测试方案，方便按里程碑推进。

## 交付物

最终交付 **4 份 Markdown 文档**，作为 Coding Agent 的开发指南：

1. 需求文档（PRD）
2. 技术架构设计
3. API 接口设计
4. 任务清单（含优先级、交付物、测试方案）（重点）

> 你只需要与 B4VC 沟通需求文档，其余内容 B4VC 会独立完善。  
> Coding Agent 从任务清单入手，按优先级参考另外三份文档开发，并在每个任务结束时交付指定内容，即可顺利推进 Vibe Coding。  
> 如果你愿意，也可以全程参与方案制定。
